🚀 SSH Key unter Windows erstellen + Ubuntu‑Server Verbindung

Windows Terminal · OpenSSH · Ubuntu 22.04/24.04
⚡ Praxisanleitung: Sicherer Serverzugriff ohne Passwort
Du lernst, wie du unter Windows moderne SSH‑Schlüssel (Ed25519 & RSA 4096 Bit) generierst, den öffentlichen Schlüssel auf deinen Ubuntu‑Server importierst und dich mit einer einzigen Terminal‑Zeile verbindest – vollständig verschlüsselt und komfortabel. Inklusive Sudo‑User Vorbereitung & Server‑Hardening.
1 🔹 Modernen SSH‑Key (Ed25519) generieren

Öffne Windows Terminal, PowerShell oder CMD. Der ed25519 Algorithmus bietet beste Sicherheit bei hoher Geschwindigkeit – ideal für moderne Server.

ssh-keygen -t ed25519 -C "windows-benutzer@mein-rechner"

Nach Eingabe wirst du gefragt:

  • Speicherort: Drücke Enter (Standardpfad C:\Users\DEIN_NAME\.ssh\id_ed25519)
  • Passphrase: Optional, aber empfohlen (zusätzlicher Schutz)
💡 Hinweis: Nach der Generierung erhältst du zwei Dateien: id_ed25519 (PRIVAT – geheim halten) und id_ed25519.pub (ÖFFENTLICH – für den Server).
2 🔸 RSA 4096 Bit – universell einsetzbar

Falls dein Server ältere SSH‑Versionen nutzt oder du einen breit kompatiblen Schlüssel benötigst, generiere einen RSA‑Schlüssel mit maximaler Bitlänge.

ssh-keygen -t rsa -b 4096 -C "windows-rsa-key@user"

Speicherort: Gib ggf. einen individuellen Namen an, z.B. ~/.ssh/id_rsa_4096.
Tipp: Du kannst mehrere Schlüssel problemlos nebeneinander verwalten.

✅ Nach der Erstellung findest du beide Schlüsselpaare unter %USERPROFILE%\.ssh\ – perfekt für verschiedene Server oder Dienste.
3 📋 Öffentlichen Schlüssel auslesen (Kopieren für den Server)

Damit der Ubuntu‑Server deinen Zugriff erlaubt, benötigt er den öffentlichen Schlüssel. Kopiere ihn mit einem der folgenden Befehle:

# PowerShell (Ed25519 Beispiel) Get-Content $env:USERPROFILE\.ssh\id_ed25519.pub | Set-Clipboard # oder klassisch anzeigen: type $env:USERPROFILE\.ssh\id_ed25519.pub

Für den RSA‑Schlüssel entsprechend: id_rsa_4096.pub.

📌 Der Inhalt beginnt mit ssh-ed25519 AAAAC3... oder ssh-rsa AAAAB3... – diesen gesamten Text wirst du auf dem Ubuntu‑Server in die authorized_keys Datei einfügen.
4 🐧 Ubuntu‑Server: Public Key importieren (sudo‑User)

Voraussetzung: Du hast SSH‑Zugang zu deinem Ubuntu‑Server (z.B. mit Passwort oder existierendem Key).
Wir bereiten alles vor, damit dein Windows‑Rechner sich später passwortlos verbinden kann.

1. Auf dem Ubuntu‑Server einloggen (z.B. mit Passwort):

ssh benutzername@203.0.113.10

2. Sicherstellen, dass der .ssh Ordner existiert (falls nicht, anlegen):

mkdir -p ~/.ssh chmod 700 ~/.ssh

3. Öffentlichen Schlüssel aus Windows in authorized_keys eintragen:
(Kopiere den zuvor kopierten Public‑Key aus deiner Windows‑Zwischenablage)

echo "ssh-ed25519 AAAAC3......dein-key" >> ~/.ssh/authorized_keys # oder mit nano/editor: nano ~/.ssh/authorized_keys

4. Berechtigungen korrigieren (wichtig für Sicherheit):

chmod 600 ~/.ssh/authorized_keys
🚀 Sudo‑User Optimierung: Falls dein Benutzer sudo‑Rechte besitzt, kannst du den Key auch für root‑ähnliche Tasks nutzen. Zusätzlich kannst du die SSH‑Server‑Konfiguration prüfen: sudo nano /etc/ssh/sshd_config – Stelle sicher, dass PubkeyAuthentication yes aktiv ist. Danach SSH‑Dienst neustarten: sudo systemctl restart sshd

✅ Nach diesem Schritt akzeptiert der Ubuntu‑Server deinen Windows‑SSH‑Key – du bist bereit für die Verbindung!

5 🌐 Windows Terminal → Verbindung zum Ubuntu‑Server (203.0.113.10)

Nachdem der öffentliche Schlüssel auf dem Server hinterlegt ist, verbindest du dich ganz einfach ohne Passwort:

ssh benutzername@203.0.113.10

Falls du einen anderen Schlüssel verwenden möchtest (z.B. deinen RSA‑Key), nutze den -i Parameter:

ssh -i ~/.ssh/id_rsa_4096 benutzername@203.0.113.10

Tipp: Mit -v (verbose) siehst du detaillierte Verbindungsinformationen zur Fehlersuche.

🔥 Beispiel für eine erfolgreiche Anmeldung:
Willkommensnachricht erscheint und du erhältst eine Shell auf deinem Ubuntu‑Server – ohne Passwortabfrage! Du kannst sofort sudo apt update oder ls -la ausführen.
📍 Server‑IP in diesem Guide: 203.0.113.10 (Beispiel‑Adresse – ersetze durch deine echte Server‑IP).
⚙️ SSH‑Agent unter Windows – Passphrase speichern

Damit du die Passphrase nicht jedes Mal neu eingeben musst, nutze den integrierten OpenSSH‑Agent:

# SSH-Agent Dienst starten (PowerShell als Administrator) Set-Service -Name ssh-agent -StartupType 'Automatic' Start-Service ssh-agent # Privaten Schlüssel hinzufügen (z.B. ed25519) ssh-add $env:USERPROFILE\.ssh\id_ed25519 # Alle hinzugefügten Schlüssel anzeigen ssh-add -l

Nach dem ssh-add fragt der Agent nicht mehr nach der Passphrase – perfekt für Workflows mit vielen Verbindungen.

✨ Windows speichert den Schlüssel im Agent bis zur Abmeldung. Du kannst auch ssh-add -K (Windows-spezifisch) verwenden, um die Passphrase im verschlüsselten Windows‑Credential‑Manager zu hinterlegen.
🛠️ Fehlerbehebung & Profi‑Tipps
  • 🔐 „Permission denied (publickey)“ – Prüfe auf dem Ubuntu‑Server die Rechte: ls -la ~/.ssh/ – authorized_keys muss 600 sein, .ssh Ordner 700.
  • 🔍 Falscher Schlüssel? Nutze ssh -Tv benutzername@203.0.113.10, um zu sehen, welcher Key gesendet wird.
  • 🧩 Mehrere Keys für verschiedene Server: Erstelle eine config Datei unter %USERPROFILE%\.ssh\config (ohne Endung) mit Inhalt:
    Host ubuntu-prod HostName 203.0.113.10 User dein-benutzername IdentityFile ~/.ssh/id_ed25519 IdentitiesOnly yes
    Danach reicht ssh ubuntu-prod für die Verbindung.
  • 🔄 Server weigert Ed25519? In /etc/ssh/sshd_config sollte PubkeyAcceptedAlgorithms +ssh-ed25519 stehen (bei älteren Distros).
  • ⚡ Windows OpenSSH nicht installiert? Gehe zu Einstellungen → Apps → Optionale Features → OpenSSH Client installieren. Auch in Git Bash verfügbar.
⚠️ Wichtiger Sicherheitshinweis: Der private Schlüssel (id_ed25519 / id_rsa) darf niemals an Dritte oder in Repositories gelangen. Verwende immer eine Passphrase!
📌 Quick‑Reference: Dein Windows → Ubuntu SSH Workflow
🔹 Ed25519 Key generieren: ssh-keygen -t ed25519 -C "mein-server-key"
🔹 RSA 4096 Key generieren: ssh-keygen -t rsa -b 4096
🔹 Öffentlichen Key anzeigen & kopieren: type ~/.ssh/id_ed25519.pub (dann manuell kopieren)
🔹 Auf Ubuntu importieren (vom Windows aus): type ~/.ssh/id_ed25519.pub | ssh benutzer@203.0.113.10 "cat >> ~/.ssh/authorized_keys"
(Einzeiler – perfekt!)
🔹 Verbindung herstellen: ssh benutzername@203.0.113.10
💡 Profi‑Tipp: Direktes Kopieren des Public Keys per Befehl (ohne manuelles Einfügen)
Get-Content ~/.ssh/id_ed25519.pub | ssh benutzer@203.0.113.10 "umask 077; test -d .ssh || mkdir .ssh ; cat >> .ssh/authorized_keys"
(PowerShell – überträgt den Key in einem Schritt auf den Ubuntu‑Server)
🧪 Teste die Verbindung nach dem Import: ssh -o PreferredAuthentications=publickey benutzername@203.0.113.10 – so wird nur pubkey authentifizierung erzwungen.

🛡️ Empfohlene Server‑Hardening (nach Key‑Import)

  • 🔒 Deaktiviere Passwort‑Login auf dem Ubuntu‑Server (sobald der SSH‑Key funktioniert): sudo sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
  • 🚫 Root‑Login über SSH verbieten: PermitRootLogin prohibit-password (oder no)
  • 📡 Ändere den SSH‑Port (optional): Port 2222 – dann Verbindung mit ssh -p 2222 user@203.0.113.10
  • 🧾 Nutze fail2ban auf dem Ubuntu‑Server für zusätzliche Absicherung gegen Brute‑Force.
🎯 Nach Abschluss dieser Anleitung arbeitest du mit einem hochsicheren, passwortfreien SSH‑Zugang zwischen Windows und deinem Ubuntu‑Server – ideal für DevOps, Homelab oder Cloud‑Infrastruktur.
