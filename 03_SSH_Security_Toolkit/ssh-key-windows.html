🔐 SSH Key unter Windows erstellen

OpenSSH · Git Bash · PowerShell
📘 Warum SSH-Schlüssel? Mit einem SSH-Schlüsselpaar (öffentlich / privat) kannst du dich sicher gegenüber Servern, Git-Diensten (GitHub, GitLab, Bitbucket) oder Remote-Rechnern authentifizieren – ohne Passwort. Diese Anleitung zeigt dir Schritt für Schritt, wie du unter Windows SSH-Keys generierst, verwaltest und nutzt.
1 Modernen SSH-Schlüsselpaar generieren (Ed25519 / ECDSA)

Für modernste Sicherheit und bessere Performance wird Ed25519 empfohlen (falls vom Server unterstützt):

ssh-keygen -t ed25519 -C "dein-kommentar@example.com"

Weitere Varianten:

ECDSA (256/384/521): ssh-keygen -t ecdsa -b 521 DSA (veraltet, nicht empfehlenswert)
✅ Nach dem Befehl erhältst du zwei Dateien: id_rsa (privat) und id_rsa.pub (öffentlich).
2 SSH-Schlüsselpaar generieren (ssh-keygen)

Öffne PowerShell, Eingabeaufforderung (CMD) oder Git Bash. Standardmäßig wird ein 3072-bit RSA Schlüssel erstellt (sehr sicher & kompatibel). Gib ein:

ssh-keygen -t rsa -b 4096 -C "deine.email@example.com"

Du wirst nach dem Speicherort gefragt. Drücke Enter um den Standardpfad C:\Users\DEIN_BENUTZER\.ssh\id_rsa zu verwenden.
Tipp: Optional kannst du eine Passphrase vergeben – das ist ein zusätzliches Sicherheitsmerkmal.

🔒 Passphrase: Wenn du eine Passphrase festlegst, wird der private Schlüssel verschlüsselt abgelegt. Du musst sie dann bei jeder Verwendung eingeben (kannst sie aber auch im SSH-Agent speichern).
3 .ssh Ordner & Dateien überprüfen

Deine Schlüssel liegen standardmäßig in:

%USERPROFILE%\.ssh\

Um sie anzuzeigen (PowerShell):

dir $env:USERPROFILE\.ssh\

Typischer Inhalt:

  • id_rsa – privater Schlüssel (niemals teilen!)
  • id_rsa.pub – öffentlicher Schlüssel (kann auf Server/GitHub hochgeladen werden)
  • optional: config – SSH-Client-Konfiguration (für Hosts, Aliase, Keys)
  • known_hosts – gespeicherte Server-Fingerprints
⚠️ Achtung: Der private Schlüssel darf niemals weitergegeben werden! Bewahre ihn sicher auf.
4 Öffentlichen Schlüssel auslesen / kopieren

Um den öffentlichen Schlüssel in die Zwischenablage zu kopieren (z.B. für GitHub, GitLab, Server-Admin):

# PowerShell / CMD: type $env:USERPROFILE\.ssh\id_rsa.pub # Direkt in die Zwischenablage (Windows): Get-Content $env:USERPROFILE\.ssh\id_rsa.pub | Set-Clipboard

Alternativ mit cat ~/.ssh/id_rsa.pub in Git Bash. Der Inhalt beginnt mit ssh-rsa AAA....

📋 Tipp für GitHub: Gehe zu Settings → SSH and GPG keys → New SSH Key, füge den öffentlichen Schlüssel ein und speichere.
5 SSH-Agent nutzen (Passphrase speichern)

Der SSH-Agent verwaltet deine privaten Schlüssel im Arbeitsspeicher, damit du die Passphrase nur einmal eingibst.

SSH-Agent starten (PowerShell als Admin):

Set-Service -Name ssh-agent -StartupType 'Automatic' Start-Service ssh-agent

Privaten Schlüssel hinzufügen:

ssh-add $env:USERPROFILE\.ssh\id_rsa

Falls du mehrere Schlüssel hast, kannst du sie alle hinzufügen. Mit ssh-add -l werden geladene Schlüssel angezeigt.

✨ Nach einmaligem ssh-add fragt der Agent nicht mehr nach der Passphrase, bis du dich abmeldest oder den Agent stoppst.
6 SSH-Verbindung testen

Nachdem der öffentliche Schlüssel auf dem Zielserver (GitHub, eigener Linux-Server, etc.) hinterlegt wurde, teste die Verbindung:

ssh -T git@github.com

Erfolgreiche Antwort sollte etwa so aussehen:

Hi Benutzername! You've successfully authenticated, but GitHub does not provide shell access.

Für eigene Server nutze:

ssh user@server-ip -p 22

Du wirst direkt eingeloggt (sofern der Public Key in ~/.ssh/authorized_keys hinterlegt ist).

7 Tipps & häufige Fehlerquellen
  • ✔ Falsche Berechtigungen: Unter Windows sind die Berechtigungen für .ssh Ordner meist korrekt. Falls Probleme auftauchen, stelle sicher, dass der private Schlüssel nicht für "Jeder" lesbar ist.
  • ✔ Mehrere Schlüssel verwalten: Lege eine config Datei im .ssh Ordner an (ohne Endung). Beispiel:
    # ~/.ssh/config Host github.com HostName github.com User git IdentityFile ~/.ssh/id_ed25519_github Host myserver HostName 192.168.1.100 User admin IdentityFile ~/.ssh/id_rsa_lan
  • ✔ OpenSSH in PowerShell startet nicht? Starte den Dienst manuell über services.msc → "OpenSSH Authentication Agent" auf "Automatisch" setzen.
  • ✔ Ältere Windows-Versionen: Nutze Git Bash oder installiere OpenSSH via optional features. Gegebenenfalls ssh-keygen aus Git/USR/Bin verwenden.
💡 Empfehlung: Verwende für verschiedene Dienste unterschiedliche Schlüssel (z.B. je ein SSH-Key für GitHub, GitLab und Firmen-Server). Dazu einfach ssh-keygen -t ed25519 -f ~/.ssh/id_github und die IdentityFile in der Config angeben.
📌 Schnell-Referenz
🔹 Schlüssel generieren (RSA 4096): ssh-keygen -t rsa -b 4096
🔹 Ed25519 modern: ssh-keygen -t ed25519
🔹 Öffentlichen Schlüssel anzeigen: cat ~/.ssh/id_rsa.pub
🔹 Schlüssel zu Agent hinzufügen: ssh-add ~/.ssh/id_rsa
🔹 Verbindung testen (GitHub): ssh -T git@github.com
🧠 Weitere wichtige Befehle: ssh-keygen -l -f ~/.ssh/id_rsa.pub (Fingerabdruck prüfen) und ssh-keygen -p (Passphrase eines existierenden Schlüssels ändern).

🛡️ Best Practices für SSH-Keys unter Windows

  • 🔐 Verwende immer eine starke Passphrase für private Schlüssel, besonders auf Laptops.
  • 🗂️ Sichere deinen .ssh Ordner regelmäßig (z.B. mit verschlüsseltem Backup).
  • 🔄 Rotiere Schlüssel bei Verdacht auf Kompromittierung oder alle 12–24 Monate.
  • 📎 Füge niemals den privaten Schlüssel zu Repositories oder Cloud-Speicher hinzu (Ausnahme: Ende-zu-Ende verschlüsselte Tresore wie Bitwarden).
  • ✅ Nutze den SSH-Agent in Kombination mit Windows Hello / TPM, wenn möglich (OpenSSH unterstützt jetzt hardware-geschützte Schlüssel).
